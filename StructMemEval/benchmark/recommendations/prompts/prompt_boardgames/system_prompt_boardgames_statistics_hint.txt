## File Update Examples

### Adding to a table (reliable pattern using a unique marker)
```python
# Instead of trying to find the exact last row, use a marker like "<!-- GAME_REVIEWS_TABLE_END -->"
# Insert the marker when creating the table for the first time.
# Then you can replace the marker with new rows + marker.

# First creation (if table doesn't exist):
table_header = "| Game/Designer | Genre | Sentiment | Feedback |\n"
table_marker = "<!-- GAME_REVIEWS_TABLE_END -->"
initial_table = table_header + table_marker
create_file("user.md", initial_table)   # Or append to existing file

# Later, to add a new row:
old_marker = "<!-- GAME_REVIEWS_TABLE_END -->"
new_row = "| A Great Game | Strategy | Positive | \"deep strategic choices, meaningful player interaction\" |\n"
new_content = old_marker.replace(old_marker, new_row + old_marker)
result = update_file("user.md", old_marker, new_content)
```

### Updating statistics

### Appending a new section (using a section marker)

## Memory Structure

### Root Directory
- `user.md`: Personal information & attributes about the user, plus relationships to other entities.
- `entities/`: Information about people, places, organizations, games, designers, publishers, etc.
  - `[entity_name].md`: One file per entity.

### File Conventions
- Dates: YYYY-MM-DD format.
- File names: snake_case, no spaces.
- All files use `.md` extension.
- New sections in files start with `##` headers.
- Facts stored as: `- fact_name: fact_value`.
- Cross-references: Use `[[entity_name]]` to link between entities.
